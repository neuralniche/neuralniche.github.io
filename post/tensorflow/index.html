tensorflow 0.7.0 dockerfile

Jan 1, 0001

TensorFlow

Google just came out with a new deep learning framework/tensor library similar in many ways to Theano and I wanted to use this post to get familiar with it. A lot of simple examples have been posted so far so I would like to give an example of how to use TensorFlow on time-series data using some LSTM based model.

Background

tensorflow 0.7.0 just came out and while i had built a docker image for python3 with tensorflow, ran into this post: http://eatcodeplay.com/installing-gpu-enabled-tensorflow-with-python-3-4-in-ec2/ I had built a docker image prior to this that I believe was faster than ~75 minutes

Docker Hub Link Dockerfile

but wanted to fix it for 0.7.0 and found the blog post and figured i would write this up

Requirements

  • GPU with 3.0> cuda compute capabilities

Since I am using the grid k2, this won’t be the most efficient way to build a docker image for tesla or cuda compute capabilities >3.0 (afaik)

New Dockerfile

The new dockerfile isnt terribly different (and would translate easy to just a normal VM on amazon or what not) but requires a few changes from my original file that I am listing for my own clarity.

  • First, changing to get-pip.py install of pip3 vs apt-get install python3-pip due to version number and error with protobuf with the severely outdated ubuntu repo version.

  • Second, using the latest version of Bazel seems to work which seems smart to track along with due to it being a google product that I am somewhat assuming the tensorflow team uses a recent version of.

  • Third, setting env so config builds. Straight forward using the official nvidia dockerfile and locating required files including:

    • ENV CUDA_TOOLKIT_PATH /usr/local/cuda-7.5
    • ENV CUDNN_INSTALL_PATH /usr/lib/x86_64-linux-gnu
    • ENV TF_NEED_CUDA 1
    • ENV PYTHON_BIN_PATH /usr/bin/python3
    • ENV TF_CUDA_COMPUTE_CAPABILITIES “3.0”
    • symlink the cudnn file required ln -s /usr/include/cudnn.h /usr/lib/x86_64-linux-gnu/cudnn.h
